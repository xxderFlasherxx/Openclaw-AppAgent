GESAMTKONZEPT
--------------------------------------------------------------------------------

Das System besteht aus drei KI-Agenten, die zusammenarbeiten:

1. HANS (OpenClaw AI / Gizmo)
   - Rolle: Orchestrator / Bauleiter / Projektmanager
   - Funktion: Steuert den gesamten Ablauf, bedient das Dateisystem,
     führt Terminal-Befehle aus, kommuniziert mit dem User über Telegram,
     koordiniert die anderen beiden Agenten.
   - Technologie: OpenClaw AI (Node.js-basiert), läuft als Daemon
   - Bereits vorhanden: Ja, bereits konfiguriert mit Telegram, Skills, etc.
   - Modell: kimi-k2.5-ollama (Cloud)

2. GÜNTHER (Ollama Cloud / Kimi K2.5)
   - Rolle: Architekt / Vermittler / Prompt-Formulierer
   - Funktion: Erstellt den 10-Phasen-Plan für das Spiel, formuliert
     perfekte Prompts für den VS Code Copilot, analysiert Fehler und
     erstellt Korrektur-Prompts. Er ist das strategische Gehirn.
   - Technologie: Ollama Cloud API (OpenAI-kompatibel)
   - Bereits vorhanden: Ja, konfiguriert unter ollama_cloud Provider
   - Modell: kimi-k2.5

3. DER COPILOT (VS Code Copilot / Claude Sonnet 4.6)
   - Rolle: Meister-Programmierer / Code-Generator
   - Funktion: Schreibt den eigentlichen C#-Code für Unity, erstellt
     Shader, konfiguriert Szenen, implementiert Game-Mechaniken.
   - Technologie: VS Code mit GitHub Copilot Extension
   - Modelle: Claude Sonnet 4.6 (Standard), Opus (für komplexe Logik),
     Haiku (für schnelle kleine Aufgaben)
   - Interaktion: Über VS Code CLI oder UI-Automation

TECHNOLOGIE-STACK
--------------------------------------------------------------------------------

Backend/Orchestrierung:
  - OpenClaw AI (v2026.3.x) mit Node.js Runtime
  - Skill-System: SKILL.md-basierte Markdown-Skills
  - Tools: exec, read, write, edit, apply_patch, web_fetch, message

      # Game Development Orchestrator

      Dieses Skill-System ermöglicht die vollautonome Entwicklung von Spielen
      in Unity. Es nutzt ein 3-Agenten-System:

      ## Architektur

      1. **Hans (OpenClaw/Gizmo)**: Orchestrator
         - Steuert den Gesamtablauf
         - Erstellt/bearbeitet Dateien
         - Führt Terminal-Befehle aus
         - Kommuniziert mit User via Telegram

      2. **Günther (Ollama Cloud / Kimi K2.5)**: Architekt
         - Erstellt 10-Phasen-Pläne
         - Formuliert Prompts für den Copilot
         - Analysiert Fehler
         - Entscheidet über Phase-Übergänge

      3. **VS Code Copilot (Claude Sonnet 4.6)**: Programmierer
         - Schreibt den eigentlichen C#-Code
         - Erstellt Unity-Scripts
         - Implementiert Game-Mechaniken

      ## Aktivierung

      Der Skill wird aktiviert wenn der User Nachrichten sendet wie:
      - "Bau mir ein Spiel..."
      - "Erstelle ein Game..."
      - "Entwickle ein Spiel wie..."
      - "Mach ein Game im Stil von..."
      - "Neues Spielprojekt..."

      ## Workflow

      ### Phase 0: Initialisierung
      1. User-Prompt empfangen und verstehen
      2. Genre erkennen (Driving, Platformer, RPG, Sandbox, etc.)
      3. Genre-spezifischen Kontext laden (aus prompts/game-genres/)
      4. An Günther senden für Planungserstellung

      ### Phase 1-9: Entwicklung
      Für jede Phase:
      1. Günther nach Phase-Prompt fragen
      2. Unity-Projektordner vorbereiten
      3. Prompt an VS Code Copilot injizieren
      4. Auf Code-Generierung warten
      5. Unity-Kompilierung prüfen
      6. Bei Fehler: Fehler-Korrektur-Schleife
      7. Bei Erfolg: Phase als abgeschlossen markieren
      8. Weiter zur nächsten Phase

      ### Phase 10: Finalisierung
      1. Unity Build erstellen
      2. Build testen (kompiliert er?)
      3. Build-Ordner bereinigen
      4. User benachrichtigen mit Ergebnis

      ## Zustandsverwaltung

      Alle Zustände werden in .plan/ Ordner im Projektverzeichnis gespeichert:
      - master-plan.json      → Der 10-Phasen-Plan von Günther
      - current-phase.json    → Aktuelle Phase + Status
      - phase-history.json    → Alle bisherigen Phasen + Ergebnisse
      - error-log.json        → Alle aufgetretenen Fehler + Lösungen
      - copilot-prompts.json  → Alle an den Copilot gesendeten Prompts

      ## Fehlerbehandlung

      - Max 5 Korrektur-Versuche pro Phase
      - Nach 5 Fehlschlägen: User benachrichtigen und um Input bitten
      - Fehler werden in error-log.json protokolliert
      - Günther analysiert Fehler und formuliert Korrektur-Prompts

      ## Sicherheit

      - Keine externen Aktionen ohne User-Bestätigung
      - Nur im vorgesehenen Projektordner arbeiten
      - Unity Build nur lokal, kein Upload
      - Alle Aktionen werden geloggt

SCHRITT 9: Sub-Skill "unity-watcher" erstellen
--------------------------------------------------------------------------------

BESCHREIBUNG:
  Dieser Sub-Skill überwacht Unity und meldet Kompilierungsfehler,
  Runtime-Fehler und den allgemeinen Projektstatus an Hans.

WAS SONNET MACHEN SOLL:

  9.1 Erstelle das Unity Editor-Script "AutoCompileWatcher.cs":

      Datei: templates/AutoCompileWatcher.cs.txt

      """
      using UnityEngine;
      using UnityEditor;
      using System.IO;
      using System.Text;

      [InitializeOnLoad]
      public class AutoCompileWatcher
      {
          static string statusFile;

          static AutoCompileWatcher()
          {
              statusFile = Path.Combine(
                  Application.dataPath, "..", ".plan", "unity-status.json"
              );

              CompilationPipeline.compilationFinished += OnCompilationFinished;
              CompilationPipeline.compilationStarted += OnCompilationStarted;
              Application.logMessageReceived += OnLogMessage;

              WriteStatus("idle", "Unity Editor bereit.");
          }

          static void OnCompilationStarted(object context)
          {
              WriteStatus("compiling", "Kompilierung gestartet...");
          }

          static void OnCompilationFinished(object context)
          {
              // Prüfe auf Fehler
              var errors = new StringBuilder();
              int errorCount = 0;

              // Unity-interne Fehler-Abfrage
              // CompilationPipeline stellt diese bereit

              if (errorCount == 0)
              {
                  WriteStatus("success", "Kompilierung erfolgreich. Keine Fehler.");
              }
              else
              {
                  WriteStatus("error", errors.ToString());
              }
          }

          static void OnLogMessage(string condition, string stackTrace,
                                    LogType type)
          {
              if (type == LogType.Error || type == LogType.Exception)
              {
                  string errorJson = JsonUtility.ToJson(new ErrorEntry
                  {
                      timestamp = System.DateTime.Now.ToString("o"),
                      message = condition,
                      stackTrace = stackTrace,
                      type = type.ToString()
                  });

                  string errorLog = Path.Combine(
                      Application.dataPath, "..", ".plan", "error-log.json"
                  );

                  // Append to error log
                  File.AppendAllText(errorLog, errorJson + "\n");

                  WriteStatus("runtime-error", condition);
              }
          }

          static void WriteStatus(string status, string message)
          {
              string json = "{"
                  + "\"status\":\"" + status + "\","
                  + "\"message\":\"" + EscapeJson(message) + "\","
                  + "\"timestamp\":\"" + System.DateTime.Now.ToString("o") + "\""
                  + "}";

              File.WriteAllText(statusFile, json);
          }

          static string EscapeJson(string s)
          {
              return s.Replace("\\", "\\\\")
                      .Replace("\"", "\\\"")
                      .Replace("\n", "\\n")
                      .Replace("\r", "\\r");
          }

          [System.Serializable]
          class ErrorEntry
          {
              public string timestamp;
              public string message;
              public string stackTrace;
              public string type;
          }
      }
      """

  9.2 Beschreibe den Watcher-Ablauf für Hans:

        c) Interpretation:
           - "idle"          → Unity hat noch nicht reagiert, warte weiter

           - "success"       → Alles OK, weiter zur nächsten Phase
           - "error"         → Kompilierungsfehler, starte Fehlerkorrektur
           - "runtime-error" → Laufzeitfehler, starte Fehlerkorrektur
        d) Hans pollt alle 3 Sekunden bis ein finaler Status erreicht ist
        e) Timeout nach 120 Sekunden → User benachrichtigen

  9.3 Beschreibe den Unity-Log-Reader:

      Falls der Unity Editor nicht läuft oder das Watcher-Script

      macOS Log-Pfad:
      ~/Library/Logs/Unity/Editor.log

      Hans kann diese Datei mit dem "read"-Tool auslesen und nach
      Zeilen suchen die mit "Error" oder "Exception" beginnen.

  9.4 Erstelle eine Referenz für häufige Unity-Fehler:

      Datei: references/common-unity-errors.txt

      """
      HÄUFIGE UNITY C# FEHLER UND LÖSUNGEN

      1. CS0246: The type or namespace name 'X' could not be found
         → using-Statement fehlt oder Package nicht installiert

      2. CS1061: 'Type' does not contain a definition for 'Member'
         → Tippfehler im Methodennamen oder falsche API-Version

      3. NullReferenceException
         → Objekt nicht zugewiesen. GetComponent<>() oder Find() fehlgeschlagen

      4. CS0029: Cannot implicitly convert type 'X' to 'Y'
         → Falscher Datentyp. Cast nötig oder Variable-Typ ändern

      5. UnityException: [Object] can only be called from the main thread
         → Async/Thread-Problem. Verwende UnityMainThreadDispatcher

      6. MissingComponentException
         → RequireComponent-Attribut fehlt oder Komponente nicht hinzugefügt

      7. StackOverflowException
         → Endlose Rekursion. Update() ruft sich möglicherweise selbst auf

      8. SerializationException
         → [Serializable]-Attribut fehlt oder nicht-serialisierbarer Typ

      9. IndexOutOfRangeException
         → Array-Index prüfen. List.Count vs Array.Length

      10. InvalidOperationException: Collection was modified
          → foreach durch for-Schleife ersetzen beim Modifizieren
      """

            → Editor-Script muss Komponenten hinzufügen

         e) "IndexOutOfRange" → Logik-Fehler
            → Severity: MAJOR
            → Copilot muss Array-Bounds prüfen

         f) Unbekannter Fehler
            → Severity: UNKNOWN
            → Günther um Analyse bitten

  15.2 Implementiere den Korrektur-Prompt-Generator:

       FUNKTION: generateCorrectionPrompt(error, sourceCode, attempt)

         "Der folgende Code hat einen Fehler:

          Code:
          [sourceCode]

          Bitte korrigiere den Fehler."

         "Der vorherige Korrekturversuch hat nicht funktioniert.
          ORIGINAL FEHLER: [errorMessage]
          NEUER FEHLER: [newErrorMessage]

          Hier ist der aktuelle Code:
          [sourceCode]

          HINWEIS: Der Fehler liegt wahrscheinlich bei [Günthers Analyse].
          Bitte korrigiere das gezielt."

       Versuch 3 (Model-Upgrade zu Opus):
         "WICHTIG: Dies ist ein schwieriger Fehler, der bereits 2x

          1. Versuch: [errror1] → [fix1] → [result1]
          2. Versuch: [error2] → [fix2] → [result2]

          AKTUELLER CODE:
          [sourceCode]

          AUFGABE: Analysiere die Fehlerhistorie und finde die
          tatsächliche Ursache. Schreibe den gesamten Code der
          Datei neu, korrekt und vollständig."

  15.3 Implementiere den "Nuclear Option":

       Wenn nach 5 Versuchen nichts klappt:

          """
          using UnityEngine;

          public class Singleton<T> : MonoBehaviour where T : MonoBehaviour
          {
              public static T Instance { get; private set; }

              protected virtual void Awake()
              {
                  if (Instance != null && Instance != this)
                  {
                      Destroy(gameObject);
                      return;
                  }
                  Instance = this as T;
                  DontDestroyOnLoad(gameObject);
              }
          }
          """

          using UnityEngine;

          using System.Collections;

          public class SceneLoader : Singleton<SceneLoader>
          {
              public void LoadScene(string sceneName)
              {
                  StartCoroutine(LoadSceneAsync(sceneName));
              }

              private IEnumerator LoadSceneAsync(string sceneName)
              {
                  AsyncOperation op = SceneManager.LoadSceneAsync(sceneName);
                  while (!op.isDone)
                  {
                      yield return null;
                  }
              }
          }
          """

SCHRITT 18: Unity Editor-Skripte für Automation
--------------------------------------------------------------------------------

BESCHREIBUNG:
  Unity Editor-Skripte laufen nur im Editor (nicht im fertigen Spiel)
  und können den Editor automatisieren - z.B. Objekte erstellen,
  Komponenten zuweisen, Szenen aufbauen.

WAS SONNET MACHEN SOLL:

  18.1 Erstelle AutoBuilder.cs:

       Dieses Script wird von Hans per Unity CLI aufgerufen um
       das Projekt zu bauen.

       """
       using UnityEditor;
       using UnityEditor.Build.Reporting;
       using System.IO;

       public class AutoBuilder
       {
           [MenuItem("Build/Build Game")]
           public static void Build()
           {
               string buildPath = Path.Combine(
                   System.Environment.CurrentDirectory, "Builds"
               );
               Directory.CreateDirectory(buildPath);

               BuildPlayerOptions options = new BuildPlayerOptions
               {
                   scenes = GetAllScenes(),
                   locationPathName = Path.Combine(buildPath, "game.exe"),
                   target = BuildTarget.StandaloneWindows64,
                   options = BuildOptions.None
               };

               BuildReport report = BuildPipeline.BuildPlayer(options);

               // Status in Datei schreiben für Hans
               string statusPath = Path.Combine(
                   System.Environment.CurrentDirectory, ".plan", "build-status.json"
               );

               if (report.summary.result == BuildResult.Succeeded)
               {
                   File.WriteAllText(statusPath,
                       "{\"status\":\"success\",\"size\":\""
                       + report.summary.totalSize + "\"}");
               }
               else
               {
                   File.WriteAllText(statusPath,
                       "{\"status\":\"failed\",\"errors\":\""
                       + report.summary.totalErrors + "\"}");
               }
           }

           static string[] GetAllScenes()
           {
               var scenes = new System.Collections.Generic.List<string>();
               foreach (var scene in EditorBuildSettings.scenes)
               {
                   if (scene.enabled)
                       scenes.Add(scene.path);
               }
               return scenes.ToArray();
           }
       }
       """

  18.2 Erstelle SceneBootstrapper.cs:

       Dieses Script erstellt automatisch die Basis-GameObjects in
       einer neuen Szene.

       """
       using UnityEditor;
       using UnityEngine;

       public class SceneBootstrapper
       {
           [MenuItem("Tools/Bootstrap Scene")]
           public static void BootstrapScene()
           {
               // GameManager
               if (Object.FindObjectOfType<GameManager>() == null)
               {
                   var gmObj = new GameObject("GameManager");
                   gmObj.AddComponent<GameManager>();
               }

               // Main Camera mit Cinemachine (falls vorhanden)
               var cam = Camera.main;
               if (cam == null)
               {
                   var camObj = new GameObject("Main Camera");
                   camObj.AddComponent<Camera>();
                   camObj.tag = "MainCamera";
               }

               // Directional Light
               if (Object.FindObjectOfType<Light>() == null)
               {
                   var lightObj = new GameObject("Directional Light");
                   var light = lightObj.AddComponent<Light>();
                   light.type = LightType.Directional;
                   lightObj.transform.rotation = Quaternion.Euler(50f, -30f, 0f);
               }

               // Event System (für UI)
               if (Object.FindObjectOfType<UnityEngine.EventSystems.EventSystem>() == null)
               {
                   var esObj = new GameObject("EventSystem");
                   esObj.AddComponent<UnityEngine.EventSystems.EventSystem>();
                   esObj.AddComponent<UnityEngine.EventSystems.StandaloneInputModule>();
               }

               Debug.Log("[SceneBootstrapper] Scene bootstrapped successfully!");
           }
       }
       """

  18.3 Erstelle AutoComponentAssigner.cs:

       Dieses Script weist Komponenten automatisch zu basierend auf
       Namenskonventionen.

       """
       using UnityEditor;
       using UnityEngine;

       public class AutoComponentAssigner
       {
           [MenuItem("Tools/Auto-Assign Components")]
           public static void AssignComponents()
           {
               // Finde alle GameObjects in der Szene
               var allObjects = Object.FindObjectsOfType<GameObject>();

               foreach (var obj in allObjects)
               {
                   // Wenn ein Script mit dem gleichen Namen existiert,
                   // weise es automatisch zu
                   string scriptName = obj.name + "Controller";
                   var scriptType = System.Type.GetType(scriptName);

                   if (scriptType != null && obj.GetComponent(scriptType) == null)
                   {
                       obj.AddComponent(scriptType);
                       Debug.Log($"[AutoAssign] {scriptName} → {obj.name}");
                   }
               }
           }
       }
       """

VS CODE COPILOT STEUERUNGS-METHODEN
====================================

METHODE 1: Agent Mode (VS Code 2025+)
--------------------------------------
- Copilot hat seit 2025 einen "Agent Mode"
- Kann über VS Code Chat aufgetragen werden
- Arbeitet autonom: erstellt/bearbeitet Dateien, führt Befehle aus
- Aufruf: Cmd/Ctrl+Shift+I → Chat öffnen → Aufgabe eingeben
- Kann auch über .github/copilot-instructions.md gesteuert werden

.github/copilot-instructions.md Beispiel:
```
# Projekt-Anweisungen für Copilot
- Unity 2022.3 LTS Projekt
- C# Code Style: PascalCase für öffentliche Members
- Verwende URP (Universal Render Pipeline)
- Namespace: {GameName}
```

METHODE 2: Inline Chat
-----------------------
- Cmd/Ctrl+I → Inline-Prompt an aktueller Cursor-Position
- Generiert Code direkt in der Datei
- User muss Accept/Reject klicken

METHODE 3: Copilot CLI (gh copilot)
------------------------------------
- gh copilot suggest -t code "[prompt]"
- gh copilot explain "[code]"
- Requires: gh Extension installiert

METHODE 4: Datei-basiert (unser Hauptansatz)
---------------------------------------------
- Hans erstellt Dateien mit detaillierten Kommentaren
- Copilot vervollständigt den Code basierend auf Kommentaren
- Kein UI-Zugriff nötig
- Funktioniert zuverlässig wenn Kommentare gut formuliert sind

METHODE 5: .copilot-prompt.md
------------------------------
- Datei im Projektordner mit Anweisungen
- Copilot berücksichtigt diese bei Vorschlägen
- Gut für projektweite Anweisungen

FÜR UNSER SYSTEM EMPFOHLEN:
- Primär: Methode 4 (Datei-basiert) → Hans schreibt Dateien mit Prompts
- Sekundär: Methode 1 (Agent Mode) → wenn UI-Automation verfügbar
- Fallback: Hans generiert Code selbst über Günther (Ollama Cloud)

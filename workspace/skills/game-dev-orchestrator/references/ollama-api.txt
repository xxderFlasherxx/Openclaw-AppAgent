OLLAMA CLOUD API REFERENZ (für Hans/OpenClaw)
==============================================

Endpunkt: https://ollama.com/v1/chat/completions
Auth: Bearer [API_KEY aus openclaw.json → models.providers.ollama_cloud.apiKey]
Format: OpenAI-kompatibel

Beispiel-Aufruf (den Hans über web_fetch machen kann):

POST https://ollama.com/v1/chat/completions
Headers:
  Authorization: Bearer [API_KEY]
  Content-Type: application/json
Body:
  {
    "model": "kimi-k2.5",
    "messages": [
      {"role": "system", "content": "[System Prompt]"},
      {"role": "user", "content": "[User Prompt]"}
    ],
    "temperature": 0.3,
    "response_format": {"type": "json_object"}
  }

WICHTIG: response_format: json_object erzwingt JSON-Ausgabe
WICHTIG: temperature 0.3 für konsistente, technische Antworten

Verfügbare Modelle:
- kimi-k2.5 (Standard, 200k Kontext, kostenlos)

Fehlerbehandlung:
- HTTP 200: Erfolg → Antwort in choices[0].message.content
- HTTP 401: API-Key ungültig
- HTTP 429: Rate Limit → 30 Sekunden warten und erneut versuchen
- HTTP 500+: Server-Fehler → 10 Sekunden warten und erneut versuchen

Test-Request (um Verbindung zu prüfen):
  System: "Antworte nur mit validem JSON."
  User: "Antworte mit: {\"status\": \"ok\", \"message\": \"Günther bereit\"}"
